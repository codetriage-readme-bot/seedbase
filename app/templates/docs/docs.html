The API

The examples in the documentation are cURL statements. You can run the statements on a command line to try out different API requests. To learn more, see the cURL website.

Authentication

Basic authentication is supported with an email address and an API token. You don't need to supply authentication credentials for most endpoints, but doing so will increase your rate limit.

Use the following authentication format with your email address and API token:

{email_address}/token:{api_token}

Don't forget to url-encode the slash character if you're authenticating over HTTP!

Rate Limits

The API has a specific rate limit per month depending on your plan. See the table below:

Plan Requests per minute
No Plan 1,000
Personal 10,000
Business 100,000
Corporate Unlimited

Request Format

Since the Hatch Data API is a JSON API, you must supply a Content-Type: application/json header in PUT and POST requests and set an Accept: application/json header on all requests.

Response Format

Successful responses will be in the 200 or 300 range.
